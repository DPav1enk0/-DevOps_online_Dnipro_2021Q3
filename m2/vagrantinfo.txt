Остановка виртуальной машины
"Заморозка времени", ресурсоемкий, после остановки
остаются зарезервированными ресурсы оперативной памяти,
жесткого диска, процессора.

vagrant suspend

Соответствует выключению обычной физической машины.
Результаты работы записываются на виртуальный жесткий диск,
а все остальные ресурсы, оперативная память, процессор, возвращаются
хост-системе.

vagrant halt

В отличии от vagrant suspend, потребуется пройти полный цикл
старта операционной системы.

В случае изменения конфигурации, заново пересоздаёт виртуалку

vagrant provision

Полностью освободить зарезервированную под виртуальную машину
память

vagrant destroy

Все следы присутствия в системе будут уничтожены.
При этом образ, который был загружен при установке, не удаляется, т.к. он может быть использован другими виртуальными машинами.

Перезапустить (Например после редактирования Vagrantfile)

vagrant reload

Соединение с виртуальной машиной

vagrant ssh

Попадаем в виртуальную машину, команды
uptime
free
df -h

Покинуть виртуальную машину

exit

Добавление образа без инициализации новой виртуальной машины

vagrant box add hashicorp/bionic64

Просмотр образов

vagrant box list

Удаление ранее загруженного образа

vagrant box remove hashicorp/bionic64 virtualbox

Создать hello.txt в общей папке по умолчанию,
/vagrant/ а в хост-системе появится файл в корне проекта.

vagrant up
vagrant ssh
cd /vagrant/
touch hello.txt
ls -la
